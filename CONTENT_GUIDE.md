# Content Guide

How to add or change learning rounds (prompt + images).

## Image specs

- **Format:** PNG
- **Color:** grayscale, black-on-white art. Avoid lessons that depend on
  color differences because e-ink contrast makes them hard to identify.
- **Size:** roughly square, ~600×600 px is plenty for a 1072×1448 px
  screen split into 3–4 tiles. Larger files just slow down loading.
- **Naming:** lowercase, no spaces, e.g. `cat.png`, `apple.png`, `1.png`

## Folder layout

Images live under `plugins/toddlerlearn.koplugin/assets/<category>/`.
Current image folders include `animals/`, `fruit/`, `numbers/`, `letters/`,
`shapes/`, `vehicles/`, `body/`, `household/`, `emotions/`, and `counting/`.

Most assets are generated by `generate_assets.py`. Prefer extending that
script for new simple drawings so the art style stays consistent. The
higher-quality object icons come from AI-generated source sheets in
`asset_sources/generated_sheets/`, which the generator crops into individual
plugin assets.

## Adding a round

Rounds live inside named categories in `content.lua`:

```lua
animals = {
    label = "Animals",
    rounds = {
        {
            prompt = "Cat",                      -- text shown at the top
            answer = "animals/cat.png",          -- correct image
            distractors = {
                "animals/dog.png",
                "animals/cow.png",
            },
        },
    },
},
```

Guidelines:

- Add the category key to `Content.category_order`, otherwise it will not
  appear in mixed review or parent setup.
- **2 distractors** is the minimum. Difficulty controls how many choices
  appear: easy uses 2 total choices, normal uses 3, hard uses up to 4.
  Hard mode can derive extra distractors from other rounds in the same
  category.
- Pick distractors from the *same category* as the answer so the round
  is a meaningful choice (e.g. don't pair "Cat" with a number and a
  spoon — pair it with other animals). `letter_words` is the intentional
  exception because it teaches letter-object association. `reading_words`
  is also intentionally mixed because it teaches recognition of the prompt
  word against unlabeled pictures. `spelling_words` is not a multiple-choice
  category; it uses one picture plus scrambled letter tiles.
- Keep `prompt` short. The validator currently rejects prompts longer than
  24 characters.
- For reading practice rounds, use the target word itself as `prompt`. Keep
  picture choices unlabeled so children must read the prompt rather than
  match identical text.
- For spelling practice rounds, set `kind = "spelling"`, add a lowercase
  `word`, and provide the picture path in `answer`. Do not add distractors.
  The answer boxes are tappable and clear the current attempt.
- For text-only matching, set `kind = "text_choice"`, `answer_text`, and at
  least two `distractors_text` values. This is used for letter pairs.
- Beginning-sound rounds use a one-letter prompt and store `sound_word` so
  tests can verify that the chosen picture starts with the displayed letter.
- Leveled spelling rounds use `level = 1` for simple CVC words and higher
  values for later material; keep category rounds ordered by level.
- Word-family text rounds store the expected ending in `family`; answer text
  must share that ending and distractors must not.

## Where to get images

- Draw simple shapes yourself (black-on-white SVGs exported as PNG work
  great and dither cleanly).
- Open-licensed icon sets (e.g. simple line-art icon packs) are a good
  source of consistent, high-contrast images.
- Your own photos work too, but simplify/crop them — busy photographic
  backgrounds look noisy on e-ink and can confuse a toddler.

## Quick checklist when adding a new category

- [ ] Create `assets/<category>/` folder
- [ ] Add at least 3–4 images (1 answer + distractors per round, reused
      across rounds)
- [ ] Add category metadata and rounds to `content.lua`
- [ ] Add the category key to `Content.category_order`
- [ ] Run `python3 generate_assets.py` if assets come from the generator
- [ ] Run `./run-tests.sh`; this executes `Content.validate(...)` and
      checks referenced assets exist
- [ ] Smoke-test on the Kindle after syncing the plugin
